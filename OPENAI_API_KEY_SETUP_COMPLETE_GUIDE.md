## 1. OpenAI APIキーの取得

### ステップ1.1: OpenAIアカウントの作成

1. **OpenAI公式サイトにアクセス**
   - URL: https://platform.openai.com/signup
   - または https://openai.com/ から「API」→「Sign up」をクリック

2. **アカウント作成方法を選択**
   - Googleアカウントでサインアップ（推奨）
   - メールアドレスでサインアップ
   - Microsoftアカウントでサインアップ

3. **メール認証**
   - メールアドレスで登録した場合、確認メールが届きます
   - メール内のリンクをクリックして認証を完了

4. **基本情報の入力**
   - 名前
   - 組織名（個人の場合は個人名でOK）
   - 用途（開発/研究/ビジネスなど）

### ステップ1.2: 支払い情報の登録

⚠️ **重要**: APIキーを作成する前に、支払い情報を登録する必要があります。

1. **ダッシュボードにアクセス**
   - URL: https://platform.openai.com/account/billing

2. **支払い方法を追加**
   - 左メニューから「Billing」→「Payment methods」をクリック
   - 「Add payment method」をクリック
   - クレジットカード情報を入力（VISA、Mastercard、American Expressなど）

3. **初回チャージ（オプションだが推奨）**
   - 「Add to credit balance」から初回チャージ額を選択
   - 推奨: $5〜$10（約750円〜1,500円）
   - これにより、API使用量が即座に反映され、エラーが発生しにくくなります

### ステップ1.3: APIキーの作成

1. **APIキー管理ページにアクセス**
   - URL: https://platform.openai.com/api-keys
   - または、ダッシュボードから「API keys」をクリック

2. **新しいAPIキーを作成**
   - 「Create new secret key」ボタンをクリック
   - キー名を入力: 例「PARTS HUB Production」
   - 権限設定（新しいUIの場合）:
     - ✅ All（すべての権限）を選択（推奨）
     - または、必要な権限のみ選択:
       - ✅ Model capabilities: Read/Write
       - ✅ API: Read/Write

3. **APIキーをコピー**
   - 生成されたAPIキーをコピー（`sk-proj-` で始まる長い文字列）
   - ⚠️ **重要**: このキーは一度しか表示されません
   - 📝 **すぐにメモ帳などに保存してください**

```
例: sk-proj-ABCDefghIJKLmnopQRSTuvwxYZ1234567890abcdefGHIJKLMNOPqrstUVWXyz
```

4. **安全な保管**
   - APIキーは絶対に公開しないでください
   - GitHubやSNSに投稿しないでください
   - 他人と共有しないでください

## 2. Cloudflare Pages での環境変数設定

### 方法A: Cloudflareダッシュボード（ブラウザ）で設定【推奨・初心者向け】

#### ステップ2.1: Cloudflareダッシュボードにログイン

1. **Cloudflareにアクセス**
   - URL: https://dash.cloudflare.com/
   - メールアドレスとパスワードでログイン

#### ステップ2.2: プロジェクトを選択

1. **左メニューから「Workers & Pages」を選択**
   - 画面左側のメニューから探します

2. **「parts-hub」プロジェクトをクリック**
   - プロジェクト一覧から「parts-hub」を見つけてクリック

#### ステップ2.3: 環境変数を追加

1. **「設定」タブを開く**
   - プロジェクトページ上部の「設定（Settings）」タブをクリック

2. **「環境変数」を選択**
   - 左側のメニューから「環境変数（Environment variables）」を選択

3. **「変数を追加」をクリック**
   - 画面右上の「変数を追加（Add variable）」ボタンをクリック

4. **環境変数の情報を入力**
   ```
   変数名（Variable name）: OPENAI_API_KEY
   値（Value）: （先ほどコピーしたAPIキーを貼り付け）
   環境（Environment）: ✅ Production にチェック
   暗号化（Encrypt）: ✅ 必ずチェックを入れる
   ```

5. **「保存（Save）」をクリック**

6. **確認**
   - 環境変数一覧に以下のように表示されます:
   ```
   OPENAI_API_KEY = ********** (Encrypted, Production)
   ```

#### ステップ2.4: デプロイメントの再実行（必要な場合）

環境変数を追加した後、最新のデプロイメントが新しい変数を使用していない場合があります。

1. **「デプロイメント」タブを開く**
2. **最新のデプロイメントの右側にある「...」（三点リーダー）をクリック**
3. **「Retry deployment」を選択**
   - これにより、新しい環境変数が適用されます

### 方法B: wrangler CLI で設定【上級者向け】

⚠️ この方法は、Macにターミナルとコマンドラインツールを使い慣れている方向けです。

#### 事前準備: Node.js のインストール

```bash
# Homebrewをインストール（未インストールの場合）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Node.jsをインストール
brew install node

# インストール確認
node --version
npm --version
```

#### APIキーの設定

```bash
# プロジェクトディレクトリに移動
cd /path/to/your/webapp

# シークレットを追加
npx wrangler pages secret put OPENAI_API_KEY --project-name parts-hub

# プロンプトが表示されたら、APIキーを貼り付けてEnter
```

## 3. 動作確認

### ステップ3.1: コラム管理画面にアクセス

1. **管理画面を開く**
   - URL: https://parts-hub-tci.com/admin/articles
   - ページが正常に表示され、テーブルが読み込まれることを確認

### ステップ3.2: AI自動生成をテスト（テキストのみ）

1. **「AI自動生成」ボタンをクリック**
   - 緑色のボタン

2. **トピックを入力**
   - 例: 「ブレーキパッドの選び方」

3. **カテゴリを選択**
   - 例: 「パーツガイド」

4. **「生成する」をクリック**
   - 10〜20秒待つ
   - 編集モーダルが開き、生成された記事が表示されれば成功！

### ステップ3.3: 画像付き自動生成をテスト

1. **「自動生成（画像付き）」ボタンをクリック**
   - 青色のボタン

2. **確認ダイアログで「OK」をクリック**
   - 料金に関する警告が表示されます（約$0.05）

3. **生成を待つ**
   - 30〜60秒かかります（テキスト生成 + 画像生成）
   - ボタンが「生成中...」と表示されている間は待機してください

4. **成功メッセージを確認**
   - 「記事が生成され、公開されました！」というアラートが表示されます
   - タイトルとカテゴリが表示されます

5. **トップページで確認**
   - URL: https://parts-hub-tci.com/
   - ページを再読み込み
   - 「PARTS HUBニュース」セクションに新しい記事が表示されているはずです

## 4. トラブルシューティング

### ❌ エラー: "OpenAI APIキーが設定されていません"

**原因**: 環境変数が正しく設定されていない

**解決方法**:
1. Cloudflareダッシュボードで環境変数を再確認
   - 変数名が「OPENAI_API_KEY」（大文字）になっているか確認
   - スペースや特殊文字が入っていないか確認

2. 環境変数を削除して再作成
   - 既存の変数を削除
   - 新しく変数を追加（APIキーを再度貼り付け）

3. デプロイメントを再実行
   - 「デプロイメント」タブ → 「Retry deployment」

### ❌ エラー: "OpenAI API呼び出しに失敗しました"

**原因1**: APIキーが無効または期限切れ

**解決方法**:
1. OpenAI ダッシュボードでAPIキーを確認
   - URL: https://platform.openai.com/api-keys
   - キーが有効（Active）になっているか確認
   - 無効な場合は新しいキーを作成

**原因2**: 支払い情報が登録されていない、または残高不足

**解決方法**:
1. 支払い設定を確認
   - URL: https://platform.openai.com/account/billing
   - クレジットカードが登録されているか確認
   - 残高があるか確認（最低$5推奨）

2. 初回チャージを実行
   - 「Add to credit balance」から$5〜$10をチャージ

**原因3**: APIレート制限

**解決方法**:
1. 1〜2分待ってから再試行
2. 使用量を確認
   - URL: https://platform.openai.com/usage

### ❌ エラー: "コラム管理ページが読み込み中のまま"

**原因**: ブラウザのキャッシュまたはJavaScriptエラー

**解決方法**:
1. ページを完全に再読み込み
   - Mac: `Command + Shift + R`
   - Windows: `Ctrl + Shift + R`

2. ブラウザのキャッシュをクリア
   - Chrome: 設定 → プライバシーとセキュリティ → 閲覧履歴データの削除
   - Safari: 開発 → キャッシュを空にする

3. ブラウザのコンソールを確認
   - 右クリック → 「検証」→「Console」タブ
   - エラーメッセージがあればスクリーンショットを撮る

### ❌ エラー: "Rate limit exceeded"

**原因**: API使用量の制限に達した

**解決方法**:
1. 1〜2分待ってから再試行
2. OpenAIダッシュボードで使用量を確認
   - URL: https://platform.openai.com/usage
3. 必要に応じてアカウントのアップグレード

## 5. 料金について

### 使用するモデルと料金

#### GPT-4o-mini（テキスト生成）
- **用途**: 記事の本文生成
- **料金**: 約$0.01/記事
- **詳細**:
  - Input: $0.150 / 1M tokens
  - Output: $0.600 / 1M tokens
  - 1記事あたり約2,000トークン（入力500 + 出力1,500）

#### DALL-E 3（画像生成）
- **用途**: アイキャッチ画像生成
- **料金**: 約$0.04/画像
- **詳細**:
  - 1792×1024サイズ: $0.040/画像
  - 標準品質（standard quality）

### コスト見積もり

#### 記事生成パターン

| パターン | テキスト | 画像 | 合計 |
|---------|---------|------|------|
| テキストのみ | $0.01 | - | **$0.01** |
| 画像付き（推奨） | $0.01 | $0.04 | **$0.05** |

#### 月間コスト見積もり（画像付き記事の場合）

| 記事数/月 | コスト/記事 | 月間合計 |
|-----------|-------------|----------|
| 4記事 | $0.05 | **$0.20** （約30円） |
| 8記事 | $0.05 | **$0.40** （約60円） |
| 30記事（毎日） | $0.05 | **$1.50** （約225円） |
| 60記事（1日2回） | $0.05 | **$3.00** （約450円） |

※ 為替レート: 1ドル = 150円で計算

### コスト最適化のヒント

1. **テキストのみ生成を活用**
   - 画像が不要な記事は「AI自動生成」（緑ボタン）を使用
   - コストを$0.01に抑えられる

2. **画像を再利用**
   - 生成された画像URLをコピーして保存
   - 関連する記事で同じ画像を使用

3. **手動編集を活用**
   - AI生成後、必要に応じて手動で編集
   - 不要な部分を削除してトークン数を削減

4. **バッチ生成を避ける**
   - 一度に大量の記事を生成せず、必要な分だけ生成
   - APIレート制限も回避できる
